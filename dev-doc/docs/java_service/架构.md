# Java 调度端架构

>## 整体架构
   #### 技术路线
   + 采用Spring Cloud Alibaba
   + 使用gateway作为后台的网关
   + 使用hutool包作为http调用的工具包
   + 使用Kafka作为任务发布的消息队列
>## 任务封装端 （task_handle)
   + 对接前端，对前端提交的流程图数据进行处理和封装，每次前端提交的一张流程图为一个Task(任务)，其主要的任务就是把前端提交的数据封装为Task实体类并通过Kafka消息队列发送到调度端进行任务的消费。
>## 调度端 （scheduling）
   + 对接任务封装端，任务封装端发送任务过来之后对任务进行解析并调用，最终封装结果集返回到用户端。
